# ULGA S10F Static Candidate Ranking Views Implementation

## 1. Task Summary

S10F implements deterministic static/offline ranking views on top of the existing S10C raw ranking. It preserves raw traceability and applies S10E balancing policies without changing S10C scores or rebuilding the raw ranking graph.

## 6. S10E Contract Implemented

- raw static ranking remains untouched
- raw traceability is preserved through `raw_rank`, `raw_candidate_id`, and `raw_static_score`
- downstream consumers should use view outputs instead of raw top-N directly
- balancing is applied through deterministic view construction, not score-formula mutation

## 8. View Score Policy

S10F preserves `raw_static_score` and computes `view_score` as a deterministic downstream-only field:

- raw score base
- type-balance adjustment
- level-policy adjustment
- theme-policy adjustment
- opacity penalty
- inferred-signal penalty
- duplicate penalty

All adjustments are recorded in `balance_adjustments`. `view_score` is clamped to `0.0..1.0` and rounded to 4 decimals.

## 9. Deduplication Policy

- labels are normalized before view construction
- canonical collapse is used for balanced, A1-safe, theme-scoped, bridge, and deduplicated views
- canonical entries preserve `dedup_group_id` and `equivalent_raw_candidate_ids`
- top-20 duplicate suppression is enforced for balanced, A1-safe, and deduplicated views

## 10. A1-Safe Policy

- level ceiling remains `A1`
- pattern and vocabulary candidates are protected
- chunk candidates are capped
- opaque and advanced chunk signals are penalized or excluded
- duplicate normalized labels are removed from top-ranked windows

## 11. Theme-Scoped Policy

- theme views are built for Home, Food, School, Travel, Health, Personal, and Daily Life
- matching uses theme refs, theme metadata, chunk hints, explain hints, and keyword matching
- when available, pattern and vocabulary candidates are protected inside each theme window
- sparse theme views are reported in summary warnings

## 12. Downstream Consumer Fit

- Reading bridge: `reading_bridge_view`
- Dialogue bridge: `dialogue_bridge_view`
- Exercise or worksheet flows: `balanced_global_view` or `theme_scoped_view`
- Grammar-first flows: `pattern_first_view`
- Vocabulary-first flows: `vocabulary_first_view`

## 13. Validator Rules

The validator checks:

- top-level schema contract
- static/offline generation mode
- adaptive disabled
- required views exist
- raw traceability fields exist
- `view_score` range validity
- sequential `view_rank`
- raw score preservation
- forbidden adaptive keyword absence
- top-20 duplicate suppression for required views
- `a1_safe_view` level ceiling
- `a1_safe_view` chunk-ratio cap when enough alternatives exist
- balanced top-20 not single-type when enough mixed candidates exist
- every view candidate traces back to S10C raw ranking

## 14. Test Results

S10F adds integration tests for file generation, schema, traceability, A1 policy, deduplication, score range, raw-ranking immutability, and summary status.

## 15. Summary Result

The S10F builder produces:

- `ulga/graph/static_candidate_ranking_views.json`
- `ulga/reports/static_candidate_ranking_views_summary.json`

These outputs remain static/offline and non-adaptive.

## 16. Known Limitations

- This is not adaptive ranking.
- This does not personalize for James or Cyndi.
- This does not use learner_state.
- This does not implement Antigravity Planner.
- This does not generate Reading, Dialogue, Exercise, or Assessment content.
- This creates deterministic static views only.
