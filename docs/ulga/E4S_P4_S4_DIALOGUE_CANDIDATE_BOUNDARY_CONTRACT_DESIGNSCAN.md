## 2. Design Scope

This task defines candidate boundary and promotion rules for P4 speaking prompt records and role-play prompt packages.

This task does not implement:

- sample prompt packages
- validator code
- role-play renderer
- prompt generator
- ASR
- recording
- speech scoring
- audio pipeline
- student-facing UI
- learner-state integration
- final Dialogue Authority promotion

## 3. Source Basis

This contract extends:

- `E4S-P4-S2_SpeakingPromptContract_DesignScan`
- `E4S-P4-S3_RolePlayPromptPackageContract_DesignScan`

Inherited P4 rules:

- P4 V1 is prompt-only.
- AUX-S4 is functional sentence / oral prompt source, not full dialogue authority.
- AUX-S5 is story/context dialogue source and may preserve role/order/context.
- AUX-S7 is generated candidate only.
- P4 V1 cannot claim recording, ASR, speech scoring, audio, UI renderer, learner-state integration, production readiness, or final authority promotion.

## 4. Boundary Problem

P4 introduces two artifact layers:

1. Speaking prompt records from P4-S2.
2. Role-play prompt packages from P4-S3.

Both layers can be useful, but not all candidates have the same trust level.

The key risk is accidental promotion:

```text
raw source text
→ prompt candidate
→ generated or derived package
→ mistaken as authority
→ learner-facing final output
```

P4-S4 blocks that failure mode by defining candidate states, allowed transitions, blocked transitions, and promotion gate requirements.

## 5. Candidate Object Model

Every P4 prompt record and package should carry a boundary object.

```json
{
  "candidate_boundary": {
    "candidate_id": "BOUNDARY_SPK_AUXS5_000001",
    "applies_to_id": "SPK_AUXS5_000001",
    "applies_to_type": "speaking_prompt_record",
    "source_family": "AUX-S5",
    "candidate_origin": "source_extracted",
    "derivation_type": "none",
    "generated": false,
    "source_grounding_status": "source_grounded",
    "authority_status": "candidate_only",
    "review_status": "pending",
    "promotion_status": "not_promoted",
    "allowed_next_statuses": ["reviewed_candidate", "rejected"],
    "blocked_next_statuses": ["authority_promoted", "learner_facing_final", "production_ready"],
    "promotion_gate_results": [],
    "blocked_reasons": []
  }
}
```

### 5.1 Required boundary fields

| Field | Required | Purpose |
|---|---:|---|
| `candidate_id` | yes | Stable boundary record id. |
| `applies_to_id` | yes | Prompt or package id governed by the boundary. |
| `applies_to_type` | yes | `speaking_prompt_record` or `role_play_prompt_package`. |
| `source_family` | yes | `AUX-S4`, `AUX-S5`, or `AUX-S7`. |
| `candidate_origin` | yes | Whether it is source-extracted, derived, generated, or mixed. |
| `derivation_type` | yes | How it was derived, if applicable. |
| `generated` | yes | True if any generated content is used. |
| `source_grounding_status` | yes | Degree of source evidence. |
| `authority_status` | yes | Current authority status. |
| `review_status` | yes | Review state. |
| `promotion_status` | yes | Promotion progress state. |
| `allowed_next_statuses` | yes | Explicit allowed transitions. |
| `blocked_next_statuses` | yes | Explicit forbidden transitions. |
| `promotion_gate_results` | yes | Gate-level results. |
| `blocked_reasons` | yes | Reasons promotion is blocked. |

## 6. Controlled Enumerations

### 6.1 `applies_to_type`

Allowed values:

```text
speaking_prompt_record
role_play_prompt_package
```

### 6.2 `candidate_origin`

Allowed values:

```text
source_extracted
derived_from_source
generated_candidate
mixed_source_and_generated
manual_reviewed_candidate
```

Definitions:

| candidate_origin | Meaning |
|---|---|
| `source_extracted` | Directly extracted from source without material rewrite. |
| `derived_from_source` | Rewritten or restructured from source while preserving evidence. |
| `generated_candidate` | Generated by AI or template process; not direct authority. |
| `mixed_source_and_generated` | Contains both source-grounded and generated/derived material. |
| `manual_reviewed_candidate` | Human/reviewer accepted as a candidate, not final authority. |

### 6.3 `derivation_type`

Allowed values:

```text
none
role_label_inferred
turn_sequence_derived
functional_sentence_to_role_prompt
story_dialogue_to_prompt
story_dialogue_to_package
generated_variant
generated_context_fill
mixed_derivation
```

### 6.4 `source_grounding_status`

Allowed values:

```text
source_grounded
partially_source_grounded
derived_with_trace
generated_with_basis
generated_without_sufficient_evidence
missing_trace
```

### 6.5 `promotion_status`

Allowed V1 values:

```text
not_promoted
promotion_blocked
eligible_for_review
reviewed_candidate_ready
rejected
```

Blocked in P4 V1:

```text
authority_promoted
learner_facing_final
production_ready
```

## 7. Candidate State Machine

P4 V1 allows only candidate-level maturity movement.

```text
raw_source_reference
  ↓
source_extracted_candidate / derived_candidate / generated_candidate
  ↓
contract_valid_candidate
  ↓
eligible_for_review
  ↓
reviewed_candidate_ready
```

Alternative terminal path:

```text
candidate
  ↓
rejected
```

Blocked P4 V1 path:

```text
reviewed_candidate_ready
  ✗ authority_promoted
  ✗ learner_facing_final
  ✗ production_ready
```

Rationale:

P4 V1 is a prompt/package contract phase. It can define candidate quality and review readiness, but it must not promote records into final Dialogue Authority or learner-facing production artifacts.

## 8. Promotion Gate Rules

### Gate 0 — Source Family Gate

Pass conditions:

- `source_family` is one of `AUX-S4`, `AUX-S5`, `AUX-S7`.
- Source family matches the artifact's declared source line.

Fail conditions:

- Unknown source family.
- Artifact uses P4 fields but points to non-P4 source without explicit future approval.

### Gate 1 — Source Trace Gate

Pass conditions:

- `source_trace` exists.
- Source ids are stable or marked `TBD` only in design/example records.
- Evidence text or source pointer exists for source-grounded candidates.

Fail conditions:

- Missing source trace.
- Generated candidate claims source-grounded status without evidence.
- Package aggregates records but does not preserve package-level trace.

### Gate 2 — Origin / Derivation Gate

Pass conditions:

- `candidate_origin` is declared.
- `derivation_type` is declared.
- Generated material has `generated = true`.
- Derived material explains how it was derived.

Fail conditions:

- AUX-S7 has `generated = false`.
- Derived AUX-S4 role-play package lacks derived-role notes.
- Mixed source/generated content lacks mixed-origin marker.

### Gate 3 — Contract Shape Gate

Pass conditions:

- Speaking prompt records satisfy P4-S2 required fields.
- Role-play prompt packages satisfy P4-S3 required fields.
- Required constraint flags are present.

Fail conditions:

- Missing prompt/package required fields.
- Missing `candidate_boundary`.
- Unsupported `prompt_type`, `speaking_mode`, `package_type`, or `practice_mode`.

### Gate 4 — Source-Specific Boundary Gate

Pass conditions:

- AUX-S4 artifacts do not claim original dialogue authority.
- AUX-S5 artifacts preserve roles/order/context when available.
- AUX-S7 artifacts remain candidate-only.

Fail conditions:

- AUX-S4 claims source speaker turns when absent.
- AUX-S5 drops speaker order when available.
- AUX-S7 attempts authority promotion.

### Gate 5 — V1 Capability Boundary Gate

Pass conditions:

- Artifact remains prompt-only.
- No dependency on recording, ASR, scoring, audio, UI, or learner state.

Fail conditions:

- Claims recording support.
- Claims ASR answer checking.
- Claims pronunciation/speech score.
- Requires audio pipeline.
- Requires UI renderer.
- Requires learner-state integration.

### Gate 6 — Review Gate

Pass conditions:

- `review_status = reviewed` for reviewed candidate readiness.
- Review does not override source boundary rules.
- Reviewer notes are stored if candidate was derived/generated.

Fail conditions:

- `review_status = pending` but artifact claims reviewed readiness.
- Reviewer approves generated content as source authority without evidence.
- Review bypasses validator requirements.

### Gate 7 — Validator Gate

P4-S4 only defines this gate. Implementation waits until P4-S6.

Pass conditions after future implementation:

- P4-S2 prompt validator passes for prompt records.
- P4-S3 package validator passes for packages.
- Boundary validator passes for candidate/promotion state.

Fail conditions:

- Any validator blocking rule fails.
- Warning-only issues are treated as promotion blockers without reviewer signoff.

## 9. Allowed Transitions

### 9.1 Prompt record transitions

Allowed:

```text
candidate_only → reviewed_candidate
candidate_only → rejected
reviewed_candidate → rejected
```

Blocked in P4 V1:

```text
candidate_only → authority_promoted
reviewed_candidate → authority_promoted
reviewed_candidate → learner_facing_final
reviewed_candidate → production_ready
```

### 9.2 Package transitions

Allowed:

```text

candidate_only → rejected
reviewed_candidate → rejected
```

Blocked in P4 V1:

```text
candidate_only → production_ready
reviewed_candidate → production_ready
reviewed_candidate → learner_facing_final
reviewed_candidate → final_dialogue_authority
```

## 10. Source-Specific Promotion Rules

### 10.1 AUX-S4 Parent Functional Sentence Corpus

Maximum P4 V1 status:

```text
reviewed_candidate
```

May become reviewed candidate only if:

- It preserves functional-sentence source trace.
- It does not claim original dialogue status.
- Any role labels are marked derived.
- Any turn sequence is marked derived.
- It passes P4-S2/P4-S3 contract checks.

Must remain blocked if:

- It claims original speaker turns from source without evidence.
- It is used as full Dialogue Authority.
- It lacks source trace.
- It is mixed with generated content without mixed-origin marker.

### 10.2 AUX-S5 Story Dialogue Corpus

Maximum P4 V1 status:

```text
reviewed_candidate
```

May become reviewed candidate only if:

- Source trace exists.
- Roles are preserved if source roles exist.
- Turn order is preserved if source order exists.
- Story/context summary is grounded or marked derived.
- It passes P4-S2/P4-S3 contract checks.

Must remain blocked if:

- Speaker order is dropped without derivation marker.
- Context is flattened into isolated prompts while package claims role-play authority.
- It lacks evidence text/source pointer.
- It claims learner-facing final status.

### 10.3 AUX-S7 Generated Dialogue Candidates

Maximum P4 V1 status:

```text
candidate_only
```

AUX-S7 may be reviewed for quality, but review does not remove generated status.

May become `reviewed_candidate` only if a later task explicitly allows reviewed generated candidates and the record includes:

- `generated = true`
- generation basis
- review notes
- candidate-only guard
- no source-authority claim

Default P4-S4 rule:

```text
AUX-S7 stays candidate_only unless later explicitly approved.
```

Must remain blocked if:

- It claims source-grounded authority without evidence.
- It lacks generation metadata.
- It is marked `generated = false`.
- It claims learner-facing final or production-ready status.

## 11. Package Aggregation Rule

A package cannot exceed the weakest trust level of its components.

Rules:

1. If any child prompt is `candidate_only`, the package cannot exceed `candidate_only` unless the child is reviewed.
2. If any child prompt is AUX-S7 generated candidate, the package must carry generated or mixed-origin metadata.
3. If any child prompt lacks source trace, package promotion is blocked.
4. If the package adds derived roles or derived turns, package must declare derivation type.
5. If the package mixes AUX-S4 and AUX-S7, package must use `candidate_origin = mixed_source_and_generated`.
6. If any child prompt claims ASR/scoring/audio/UI, package promotion is blocked in V1.
