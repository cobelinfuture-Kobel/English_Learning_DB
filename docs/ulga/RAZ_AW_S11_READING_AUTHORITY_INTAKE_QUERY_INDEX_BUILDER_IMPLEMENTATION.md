# RAZ-AW-S11 Reading Authority Intake Query Index Builder Implementation

## 1. Task

`RAZ-AW-S11_ReadingAuthorityIntake_QueryIndexBuilderImplementation`

This implementation creates the first deterministic, offline Reading Authority intake query index for RAZ-derived reading candidates.

The task does **not** promote content into final Reading Authority. It creates a static intake layer that later Reading Authority, Content Query Layer, ULGA, Antigravity, and Learning Opportunity Binding stages can query.

## 5. Source Artifacts Consumed

The builder discovers existing offline JSON / JSONL files under these roots when present:

```text
work/raz/
raz_output_jsons/
output/raz/
ulga/graph/
ulga/reports/
```

It only considers files that appear to be RAZ / level-derived artifacts and excludes its own output files.

Supported intake source types:

```text
sentence_candidate
page_unit
reuse_unit_candidate
normalized_reading_unit
enriched_reading_unit
```

If no matching source artifacts are available in a local checkout, the builder still writes a stable empty index with `PASS_WITH_WARNINGS` and `no_raz_source_json_or_jsonl_files_discovered`.

## 6. Output Schema

Main output:

```text
ulga/graph/raz_reading_authority_intake_query_index.json
```

Top-level contract:

```json
{
  "schema_version": "RAZ_READING_AUTHORITY_INTAKE_QUERY_INDEX_V1",
  "builder_task": "RAZ-AW-S11_ReadingAuthorityIntake_QueryIndexBuilderImplementation",
  "generated_at": "<iso8601>",
  "source_policy": {
    "offline_static_only": true,
    "generated_content": false,
    "authority_promotion": false,
    "candidate_only_preserved": true
  },
  "levels": [],
  "items": [],
  "query_indexes": {},
  "summary": {}
}
```

Each item preserves:

- deterministic `intake_id`
- source type
- RAZ level
- book/page metadata when available
- clean text
- sentence count
- sentence candidate links when available
- source traceability
- query tags
- candidate-only authority boundary

## 7. Query Index Contract

The builder always emits these query index keys, even when some are empty:

```text

by_promotion_status
by_sentence_count_bucket
by_multi_sentence_status
by_theme_hint

```

Sentence count buckets:

```text
single_sentence
two_sentences
three_to_five_sentences
six_plus_sentences
unknown
```

## 8. Candidate-Only / No-Promotion Boundary

All intake items are emitted as:

```json
{
  "authority_status": "candidate_only",
  "promotion_status": "not_promoted",
  "generated_content": false
}
```

This is intentional.

This stage is an intake/query-index stage, not a final Reading Authority promotion stage. Reuse units, short reading seeds, writing model seeds, dialogue rewrite seeds, exercise seeds, and assessment seeds remain queryable candidates only.

## 9. Validation Contract

The S11 builder-output validator is:

```text
ulga/validators/validate_raz_reading_authority_intake_query_index_builder_output.py
```

It checks:

1. Required files exist.
2. JSON parses.
3. Top-level schema version matches.
4. Required item fields exist.
5. Intake IDs are stable and unique.
6. Query index references point only to known item IDs.
7. No item is promoted.
8. `generated_content` remains false.
9. Source traceability exists.
10. Query tag shapes are valid.
11. Sentence count bucket logic is consistent.
12. Embedded summary matches summary file.
13. Summary counts match indexed items.
14. Empty index categories are valid.

## 12. Known Limitations

- The builder uses permissive static discovery because RAZ derived artifacts may exist under multiple historical paths.
- It does not generate content.
- It does not infer missing grammar / pattern / vocabulary tags when source artifacts do not already provide them.
- It does not promote candidate readings into final Reading Authority.
- It does not bind readings to Learning Opportunity records; that belongs to a later stage.
- It does not create Reading Dependency Graph edges.
