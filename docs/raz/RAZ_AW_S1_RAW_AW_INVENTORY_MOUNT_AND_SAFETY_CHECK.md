## 7. Local PowerShell Precheck

Before running any Python audit, operator can run:

```powershell
cd G:\HomeWork\English_Learning_DB

# Confirm raw folder exists
Test-Path .\raz_output_jsons

# Count JSON files by level folder
Get-ChildItem .\raz_output_jsons -Directory | Where-Object { $_.Name -like 'Level_*' } | ForEach-Object {
  [PSCustomObject]@{
    Level = $_.Name
    JsonCount = (Get-ChildItem $_.FullName -Recurse -File -Filter *.json | Measure-Object).Count
    TotalMB = [math]::Round(((Get-ChildItem $_.FullName -Recurse -File | Measure-Object Length -Sum).Sum / 1MB), 2)
  }
} | Sort-Object Level

# Detect files over 50MB
Get-ChildItem .\raz_output_jsons -Recurse -File | Where-Object { $_.Length -gt 50MB } | Select-Object FullName, @{Name='MB';Expression={[math]::Round($_.Length/1MB,2)}} | Sort-Object MB -Descending

# Confirm raw folder is ignored by Git
git check-ignore -v raz_output_jsons/
```

Expected safety result:

```text
raz_output_jsons/ exists locally
raw files are ignored by Git
large files, if any, are reported but not staged
reports only can be committed
```

## 9. S1 Pass / Warning / Block Criteria

### PASS

```text
raw root exists
at least one A-W level folder exists
all detected raw files are JSON or explicitly allowed metadata files
no raw files are staged in Git
gitignore protects raz_output_jsons/
reports can be generated without raw text
```

### PASS_WITH_WARNINGS

```text
some levels A-W are missing
some files exceed 50MB but not staged
unexpected non-blocking metadata files exist
raw layout differs from preferred layout but can be mapped safely
```

### BLOCKED

```text
raw root does not exist
raw files are not protected by gitignore
raw files are staged in Git
media/PDF/archive files are mixed into raw JSON folder
report generation would include raw text
files over 100MB are staged or planned for commit
```
